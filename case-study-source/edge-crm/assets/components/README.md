# Case Card — Component Iterations

The four case-card design iterations from the "Behind the screens — Component
Breakdown" frame (`node 33:402`), exported in premium quality.

- **PNG** — 4× scale (1540×~532), transparent-free, crisp for slides/docs.
- **SVG** — true vector, infinitely scalable.

Each file is named by its iteration number (1 = earliest, 4 = final). See
[`../../01-case-list-story.md`](../../01-case-list-story.md) → Slide 07 for the full
critique of each.

| File | Figma node | Card sample | The lesson |
|------|-----------|-------------|-----------|
| `iteration-1.*` | `33:414` | `KLM8N12R45X` · CyberGuard Solutions · Software License · On-going · 01/02/25–01/02/25 | Led with an alphanumeric ID. No human context, no scanability. Date range ate space. Designed for the system, not the rep. |
| `iteration-2.*` | `33:439` | Android application bug review · Tantragyan Private Limited Technologies · 01/02/25 · Open · Standard | Swapped ID for a case name, but 3 metadata rows + 2 competing colours = density and cognitive overload. |
| `iteration-3.*` | `33:440` | Web dashboard loading error · Feb 10, 2025 · Techwise Ltd. · Standard · In Progress | Metadata collapsed to one line — cleaner, but users read instead of scanned; priority dominated before the case name. |
| `iteration-4.*` | `33:461` | Web dashboard loading error · Feb 10, 2025 · Techwise Ltd. · In Review · Standard Priority | **Final.** Status pill bottom-left, priority bottom-right. F-shape clicked — Why? When? Who? answered in sequence. |
